title: "Browser fingerprint spoofing for AWS registration evasion"
description: "kiro-auto randomizes Canvas, WebGL, Navigator, Screen, and Timezone browser properties to reduce bot detection during AWS account registration."
---

Modern bot-detection systems identify automated browsers not just by IP address but by the consistency of browser properties тАФ screen resolution, GPU vendor, font list, canvas rendering output, and dozens of other signals. kiro-auto's fingerprint engine randomizes these properties for each registration task, making concurrent browser sessions appear as distinct physical machines.

## Enabling and disabling

Fingerprint spoofing is **enabled by default**. To disable it, pass `--no-fingerprint`:

```bash
npm run register -- --count 3 --no-fingerprint
```

In the interactive menu, option **[6]** toggles fingerprint spoofing on or off for the current session.

<Note>
Fingerprint spoofing is designed to work alongside incognito mode (`--incognito`, enabled by default). Incognito mode prevents cookies and storage from persisting between sessions, while fingerprint spoofing ensures each incognito session presents a distinct browser identity. Using both together provides the strongest isolation between registration tasks.
</Note>

## What is spoofed

The `FingerprintGenerator` class in `lib/fingerprint/generator.ts` builds a complete `FingerprintProfile` for each task. The `FingerprintInjector` then compiles this profile into a JavaScript snippet injected into every page via Playwright's `addInitScript()` тАФ which runs before any page JavaScript тАФ so overrides are in place before the AWS registration flow begins.

### Navigator

The `navigator` object is overridden to hide automation signals and present a consistent browser identity:

| Property | Spoofed value |
|----------|--------------|
| `userAgent` | Chrome 120тАУ124 on Windows, macOS, or Linux |
| `platform` | `Win32`, `MacIntel`, or `Linux x86_64` |
| `language` | `en-US` |
| `languages` | `["en-US", "en"]` |

| `webdriver` | `false` (hides automation) |

### Canvas

`HTMLCanvasElement.prototype.toDataURL` and `CanvasRenderingContext2D.prototype.getImageData` are wrapped to inject deterministic per-pixel noise driven by a seeded random number generator. Each fingerprint profile receives a unique 16-byte hex seed, so two concurrent sessions produce measurably different canvas outputs even when rendering identical content.

### Audio

`AudioContext.prototype.createOscillator` is wrapped to add a tiny seeded frequency offset (`┬▒0.0005 Hz`) to each oscillator. This disrupts audio fingerprinting without affecting perceptible audio output.

### Timezone

`Date.prototype.getTimezoneOffset` and `Intl.DateTimeFormat` are overridden to report one of eleven real-world timezones:

```
America/New_York  America/Chicago  America/Denver  America/Los_Angeles
Europe/London     Europe/Paris     Europe/Berlin
Asia/Tokyo        Asia/Shanghai    Asia/Singapore
Australia/Sydney
```

### WebRTC

WebRTC is disabled entirely (`RTCPeerConnection` is replaced with a function that throws) to prevent IP address leaks through STUN/TURN negotiation.

### Media devices

`navigator.mediaDevices.enumerateDevices` returns a randomized but realistic list of audio inputs (1тАУ2), audio outputs (1тАУ3), and video inputs (0тАУ1) to prevent device enumeration fingerprinting.

### Fonts

Each profile receives the full OS-appropriate system font list. Windows profiles report Microsoft core fonts (Calibri, Segoe UI, etc.), macOS profiles report Apple fonts (Helvetica Neue, San Francisco, etc.), and Linux profiles report common open-source fonts (DejaVu, Liberation, Ubuntu).

## Advanced injectors

Two additional injectors run alongside the main script:

- **`clientrects-injector.ts`** тАФ wraps `Element.prototype.getBoundingClientRect` and `getClientRects` to add sub-pixel noise, disrupting layout-based fingerprinting.
- **`font-injector.ts`** тАФ patches canvas text measurement to add slight font-metric noise, making font fingerprinting less reliable.

## OS distribution

The generator selects an OS for each profile with weighted randomness that reflects real desktop market share:

- **Windows** тАФ 70 %
- **macOS** тАФ 20 %
- **Linux** тАФ 10 %

All other properties (user agent, screen resolution, WebGL GPU, font list) are generated to be internally consistent with the selected OS.
