kiro-auto is a TypeScript CLI tool that automates the full lifecycle of AWS Builder ID accounts — from batch registration using disposable email inboxes and Playwright browser automation, to injecting SSO tokens directly into Kiro IDE's credential files for instant account switching. It includes browser fingerprint spoofing, human-behavior simulation, proxy support, and Windows machine ID rotation to reduce detection.

## What kiro-auto does

kiro-auto has two primary commands:

- **`npm run register`** — Batch-registers AWS Builder ID accounts. It requests a device code from the AWS OIDC endpoint, spins up a headless Chromium browser with a randomized fingerprint, obtains a disposable email address, completes the web registration form automatically, retrieves the verification code from the inbox, and polls for the resulting OAuth refresh token. Results are written to `show/builderid-template.json`.

- **`npm run switch`** — Reads a previously saved account template, writes the refresh token and client credentials into `~/.aws/sso/cache/`, optionally resets the Windows machine ID (MachineGuid), and restarts the Kiro IDE process so the new identity takes effect immediately.

## Key features

  </Card>
  <Card title="Machine ID rotation" icon="rotate">
    Resets the Windows MachineGuid registry key to make the system appear as a new device to AWS.
  </Card>
  <Card title="Configurable concurrency" icon="bolt">
    Run multiple registration tasks in parallel with tunable concurrency, per-task delay, and proxy routing.

## Requirements

| Requirement | Version |
|---|---|
| Node.js | >= 18.0.0 |
| npm | >= 9.x |
| Operating system | Windows 10/11 recommended; macOS and Linux supported for registration |

<Warning>
Machine ID reset (`--reset-machine-id`) is only supported on Windows and requires administrator privileges.
</Warning>
